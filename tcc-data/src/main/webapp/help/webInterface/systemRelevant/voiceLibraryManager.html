企业语音库管理接口

接口作用说明

此接口可实现对企业语音库的管理，例如上传、下载、删除等

企业语音上传

接口类型：HTTP请求
接口使用账户：企业账户
接口地址：天润呼叫中心接口服务器IP或域名
接口参数：企业编号，用户名，密码
接口返回值：JSON格式返回结果
接口URL
URL示例：
http://TIWebServer:TIPort/interfaceAction/enterpriseVoiceInterface!uploadVoice.action?enterpriseId=?&userName=&pwd=&seed=&description=
参数说明
参数 要求 描述 备注
enterpriseId 必选 企业编号 管理员后台登陆后，位于右上角位置
userName 必选 用户名 登录后台用户名，如admin
pwd 必选 密码 需和seed进行MD5加密，格式：md5(md5(对应userName的密码)+seed)，例如：md5(md5(123456)seed)
seed 必选 随机字符串  
file 必选 语音文件 必须使用multipart/form-data进行post提交；如果是表单提交上传，input中的name必须为file，如；如果是代码POST提交，语音文参数名必须为file
isMoh 可选 是否为等待音 0: 否 1：是 默认为0
description 可选 文件描述  
接口使用说明
使用cURL上传语音文件：
curl -F file=@/home/haohuawuyuan.wav 'http://172.16.203.182/interfaceAction/enterpriseVoiceInterface!uploadVoice.action?enterpriseId=3010066&userName=admin&pwd=b605e86d02eef8bfd0646f6a704c17c9&seed=1234&description=uploadwav'
 
用户可以自己写一个html页面用来选择需要上传语音文件，然后以post的方式提交表单，form表单的action参数是用户上传的URL，enctype="multipart/form-data"，需要包含企业的参数验证信息，
通过<input id="file" name="file" type="file" /> 来选择上传文件,
注意：请保证name 的值与上面所描述的一致
如：
<form action="http://172.16.203.182/interfaceAction/enterpriseVoiceInterface!uploadVoice.action" enctype="multipart/form-data" method="post"> <br /> &nbsp; <p>enterpriseId: <input name="enterpriseId" type="text" /></p> <br /> &nbsp; <p>userName: <input name="userName" type="text" /></p> <br /> &nbsp; <p>pwd: <input name="pwd" type="text" /></p> <br /> &nbsp; <p>seed: <input name="seed" type="text" /></p> <br /> &nbsp; <p>file: <input name="file" type="file" /></p> <br /> &nbsp; <p>description: <input name="description" type="text" /></p> <br /> &nbsp;<input type="submit" value="Submit" />&nbsp;</form>
 
JAVA代码示例
import java.io.File;
import java.nio.charset.Charset;
 
import org.apache.http.HttpEntity;
import org.apache.http.HttpResponse;
import org.apache.http.HttpStatus;
import org.apache.http.client.HttpClient;
import org.apache.http.client.methods.HttpPost;
import org.apache.http.entity.mime.HttpMultipartMode;
import org.apache.http.entity.mime.MultipartEntityBuilder;
import org.apache.http.entity.ContentType;
import org.apache.http.impl.client.HttpClients;
import org.apache.http.util.EntityUtils;
 
public class Test {
    public static void main(String[] args) {
        String url = "http://172.16.203.182/interfaceAction/enterpriseVoiceInterface!uploadVoice.action";
 
         HttpPost httpPost = new HttpPost(url);
         try {
             HttpClient httpclient = HttpClients.createDefault();
               // 组装基本的参数
             MultipartEntityBuilder multipartEntityBuilder = MultipartEntityBuilder.create();
             multipartEntityBuilder.setMode(HttpMultipartMode.BROWSER_COMPATIBLE);
             multipartEntityBuilder.setCharset(Charset.forName("UTF-8"));
             multipartEntityBuilder.addTextBody("enterpriseId", "3010066", ContentType.create("text/plain", Charset.forName("UTF-8")));
             multipartEntityBuilder.addTextBody("userName", "admin", ContentType.create("text/plain", Charset.forName("UTF-8")));
             multipartEntityBuilder.addTextBody("pwd", "b605e86d02eef8bfd0646f6a704c17c9", ContentType.create("text/plain", Charset.forName("UTF-8")));
             multipartEntityBuilder.addTextBody("seed", "1234", ContentType.create("text/plain", Charset.forName("UTF-8")));
             multipartEntityBuilder.addTextBody("description", "描述信息", ContentType.create("text/plain", Charset.forName("UTF-8")));
      
            File file = new File("c:\\080715112220.wav");
            multipartEntityBuilder.addBinaryBody("file", file);
            HttpEntity httpEntity = multipartEntityBuilder.build();
            httpPost.setEntity(httpEntity);
            HttpResponse response = httpclient.execute(httpPost);
            // 处理响应
            if (response.getStatusLine().getStatusCode() == HttpStatus.SC_OK) {
                 // 正常返回
                 System.out.println(EntityUtils.toString(response.getEntity()));
             }else{
             }
             EntityUtils.consume(httpEntity);
         } catch (Exception e) {
             e.printStackTrace();
         }finally{
             httpPost.releaseConnection();
         }
     }
}
返回值
URL实例：
http://172.16.203.190/interfaceAction/enterpriseVoiceInterface!uploadVoice.action?enterpriseId=3000005&userName=admin&pwd=d0782bdf141e366014b4bc0d7dc6ea00&seed=aaa

error 3 企业不存在或状态不正常
error 4 无权操作，接口未开通
error 5 验证失败，用户名或密码不正确

查询企业语音

接口类型：HTTP请求
接口使用账户：企业账户
接口地址：天润呼叫中心接口服务器IP或域名
接口参数：企业编号，用户名，密码，语音名称，语音id
接口返回值：JSON格式返回结果
接口URL
URL示例：
http://TIWebServer:TIPort/interfaceAction/enterpriseVoiceInterface!listVoice.action?enterpriseId=?&userName=&pwd=&seed=&voiceName=&voiceId=
参数说明
参数 要求 描述 备注
enterpriseId 必选 企业编号 管理员后台登陆后，位于右上角位置
userName 必选 用户名 登录后台用户名，如admin
pwd 必选 密码 需和seed进行MD5加密，格式：md5(md5(对应userName的密码)+seed)，例如：md5(md5(123456)seed)
seed 必选 随机字符串  
voiceName 可选 语音名称 通过语音名称查看语音信息(名称和id都不写，则查全部)
voiceId 可选 语音id 通过语音id查看语音的信息(名称和id都不写，则查全部)
返回值
URL实例：
http://172.16.203.190/interfaceAction/enterpriseVoiceInterface!listVoice.action?enterpriseId=3000002&userName=admin&pwd=d0782bdf141e366014b4bc0d7dc6ea00&seed=aaa&voiceId=27
JSON格式数据：
{result:返回结果, code:结果代码,msg:描述,listVoice:[enterpriseId: , voiceId: , voiceName: , voicePath:,isMoh:,description: ] .......}
返回值说明：
返回值 描述
listVoice 语音列表
enterpriseId 企业编号
voiceId 语音文件Id，可用于语音文件的下载、删除接口
voiceName 语音文件名
voicePath 语音变量值，可用于设置语音变量
isMoh 0：不是等待音  1：等待音
description 描述
 
返回结果说明：
result code msg
success 0 操作成功
error 1 参数[具体参数名]不能为空
error 2 参数[具体参数名]格式不正确
error 3 企业不存在或状态不正常
error 4 无权操作，接口未开通
error 5 验证失败，用户名或密码不正确
