发送短信接口

接口作用说明

通过接口发送短信

API URL

接口URL（POST方式）
发起者：合作方
接受者：天润融通
接口类型：HTTP接口
调用方式：POST
返回格式：JSON数据
URL规范：
http://InterfaceServerIp:port/interface/sms/SendSms
例如：http://172.16.203.190/interface/sms/SendSms?hotline=4006004321&enterpriseId=3000002&userName=admin&pwd=d0782bdf141e366014b4bc0d7dc6ea00&seed=aaa&type=12&mobile=15120054657&customerName=%E7%8E%8B%E9%9D%96&msg=%E6%B5%8B%E8%AF%95&cno=2000

相关参数

参数 要求 描述 备注
hotline 可选 企业热线号码 与enterpriseId二选一
enterpriseId 可选 企业Id 与hotline二选一
userName 必选 用户名 登录后台用户名，如admin
pwd 必选 密码 对应username的密码，md5(md5(登录密码)+seed)，例如：md5(md5(123456)seed)
seed 必选 随机字符串  
type 必选 短信类型 12表示座席发送，8表示后台发送
mobile 必选 手机号 多个之间用英文逗号隔开
customerName 必选 客户姓名 多个之间用英文逗号隔开，此参数需要URLEncode
msg 必选 短信内容 此参数需要URLEncode（msg最多180个字。每条短信最多60个字，当msg超过60个字时会被自动拆分）
cno 可选 座席号 type为座席发送时使用

返回值

此接口返回JSON格式数据。
JSON格式数据
{result:返回结果,error_code:错误码, msg:结果信息}
 
result msg description
success 提交成功  
error 提交失败 参数有误、企业热线号码有误、无权操作、提交失败、短信号码错误、含有关键字等信息

示例

发送短信：
<form action="http://172.16.203.190/interface/sms/SendSms" method="post">
              <input type="text" name="hotline" value="" /></br>
              <input type="text" name="enterpriseId" value="3000002" /></br>
              <input type="text" name="userName" value="admin" /></br>
              <input type="text" name="pwd" value="d0782bdf141e366014b4bc0d7dc6ea00" /></br>
              <input type="text" name="seed" value="aaa" /></br>
              <input type="text" name="type" value="12" /></br>
              <input type="text" name="mobile" value="15120054657" /></br>
              <input type="text" name="customerName" value="王靖" /></br>
              <input type="text" name="msg" value="接口测试" /></br>
              <input type="text" name="cno" value="2000" /></br> 
              <input type="submit" value="提交" />
       </form>
 
返回：
{"result":"success","error_code":"","msg":"提交成功！"}
