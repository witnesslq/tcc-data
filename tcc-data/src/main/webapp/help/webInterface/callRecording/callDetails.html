来电通话记录详细

实现步骤

接口URL（POST方式）
发起者：合作方
接受者：天润融通
接口类型：HTTP接口
调用方式：POST
返回格式：JSON数据
URL规范：
http://InterfaceServerIp:port/interfaceAction/cdrIbInterface!listCdrIbDetail.action
说明：InterfaceServerIp:port 为使用账号登录www.ccic2.com之后，在浏览器地址栏看到的域名地址。
例如：xa.ccic2.com
           bj.ccic2.com
           1.ccic2.com

相关参数

参数 要求 描述 备注
hotline 可选 企业热线号码 与enterpriseId二选一
enterpriseId 可选 企业Id 与hotline二选一
userName 必选 用户名 登录后台用户名，如admin
pwd 必选 密码 对应username的密码，md5(md5(登录密码)+seed)，例如：md5(md5(123456)seed)
seed 必选 随机字符串  
id 必选 通话记录id 对应通话记录的uniqueId

返回值

此接口返回JSON格式数据。
JSON格式数据
{result:返回结果,
msg:{
data:[{通话记录详细数据},{}..]
}
}
 
result msg description
success {"result":"success","msg":{"data": [{"id":267,"mainUniqueId":"ccic_dev_11-1344493753.3","calleeNumber":"01041005968","clientName":"座席1","cno":"2002","startTime":"2012-08-09 14:30:08","answerTime":"2012-08-09 14:30:10","endTime":"2012-08-09 14:30:14","billDuration":"00:00:04","cost":0,"callType":"呼转座席","status":"接听" ，"sipCause":"0","record_file":"" ，"mark":""}]
}}		 成功返回通话记录详细信息
error "无权访问" 查询通话记录详细失败原因
说明： data取值为json数组。
 
json格式数据记录data的各属性定义：
名称 描述 数据格式 取值范围 例子
mainUniqueId 主通话记录唯一标识     ccic_dev_11-1344493753.3
calleeNumber 座席绑定的电话号码 区号 + 7或8位的固话号码，区号与固话号码之间无'-'；或11位长度的手机号码。 中国国内固话或手机 01041005968或18701051984
clientName 座席名称     座席1
cno 座席号     2002
startTime 开始呼叫时间     2012-08-09 14:30:08
answerTime 接听时间     2012-08-09 14:30:10
endTime 结束时间     2012-08-09 14:30:14
billDuration 计费时长     00:00:04
cost 费用     0
callType 呼叫类型     呼转座席
status 接听状态     接听
sipCause 呼叫状态 0：未识别
510：忙
511：超时
512：拒接
513：空号
514：关机
515：暂时无法接听
516：停机		    
record_file 呼叫手机录音      
mark 标识 1：呼入主叫记忆
2：外呼主叫记忆
