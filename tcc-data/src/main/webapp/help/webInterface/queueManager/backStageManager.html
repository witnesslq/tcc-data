后台管理

接口作用说明

后台管理接口主要实现：新增、修改查询座席及队列

规范

InterfaceServerIp：天润融通接口服务器Ip或域名
port：天润融通接口服务器接收端口
接口URL中的InterfaceServerIp:port地址，是登录座席前台或管理员后台在浏览器地址栏看到的域名地址，如图标注部分

座席接口-新建或修改

接口类型：HTTP
接口使用账户：企业账户
接口地址：呼叫中心系统IP
接口参数：企业账号、企业账户密码、工号、姓名等
接口返回值：操作结果
接口说明
添加一个座席。
接口URL
URL示例： http://172.16.203.197/interfaceAction/clientInterface!save.action?enterpriseId=3000008&userName=admin&password=e10adc3949ba59abbe56e057f20f883e&cno=1003&name=%E5%BA%A7%E5%B8%AD&active=1&wrapup=10&areaCode=010&power=0&callPower=0
说明：该示例中password = e10adc3949ba59abbe56e057f20f883e = MD5Encoder.encode('123456)，name值是经过URLEncode按“UTF-8”格式编码的，中文是“座席”。
参数说明
参数 要求 描述 备注
enterpriseId 必选 企业Id 例如:3000008
userName 必选 角色名称 例如:admin
password 必选 登录密码 md5(登录密码))
id 可选 座席id 如修改时为必选
cno 必选 座席工号 座席工号取值2000-9999
name 必选 座席姓名 需进行UTF-8格式的URLEncode编码
pwd 可选 座席登录密码 新增时为空则默认密码：123456,修改时为空则默认为不修改密码
crmId 可选 座席crmId 整型数字
sex 可选 性别 0表示男,1表示女；默认男
power 可选 座席角色 0表示普通座席,1表示班长座席；默认普通坐席
isSendMsg 可选 座席发送短信功能 0表示不开启,1表示开启；默认不开启
isMsgRecord 可选 座席查看短信记录功能 0表示不开启,1表示开启；默认不开启
active 可选 是否激活 0表示不激活,1表示激活；默认激活
areaCode 必选 座席所属区号 区号格式
callPower 可选 呼叫权限 0表示无限制,1表示国内长途,2表示国内本市,3表示内部呼叫；默认无限制
isOb 可选 是否可以外呼 0表示不允许,1表示允许；默认允许
ibRecord 可选 呼入是否录音 0表示不录音,1表示录音；默认录音
obRecord 可选 外呼是否录音 0表示不录音,1表示录音
wrapup 可选 座席整理时间（单位：秒） 整型数字；默认10秒
clientType 可选 座席类型 1表示电话坐席,2表示电脑座席；默认电话坐席
birth 可选 出生日期 日期类型(yyyy-MM-dd)
certificateId 可选 身份证号 整型数字
mobile 可选 移动电话 手机格式
otherTel 可选 联系电话 区号+7/8位固话号码
fax 可选 传真号码 区号+7/8位固话号码
email 可选 电子邮箱 电子邮箱格式
address 可选 联系地址 需进行UTF-8格式的URLEncode编码
post 可选 邮编  
skillIds 可选 所选的技能id 多个间用英文标点逗号间隔.例如:1,2,3
skillLevels 可选 所选的技能的等级(值越小技能越高) 多个间用英文标点逗号间隔.(与技能id相对应) 例如:5,1,1表示技能id为1的等级是5; 技能id为2的等级是1; 技能id为3的等级是1.
obClid 可选 外显号码 多个用英文逗号间隔
obClidRule 可选 外显规则 0 随机 1轮选  默认随机
obClidRecurrentselectionName 可选 轮选设置，按天或者按次更换外显号码 0 按天轮选
1 按次轮选
obClidRecurrentselectionValue 可选 轮选取值 1到30
 
返回值
JSON串，格式为{result:xxx,msg:xxx},result表示返回结果状态码，msg表示结果状态的描述。
如果成功添加座席，将返回座席id，JSON串为 {result:xxx,msg:xxx,id:xxx}
result与description属性值的定义：
result description
error 添加座席失败原因
success 成功添加座席

队列接口-新建或修改

接口类型：HTTP
接口使用账户：企业账户
接口地址：呼叫中心系统IP
接口参数：企业账号、企业账户密码、队列号、队列名、队列其他参数
接口返回值：操作结果
接口说明
添加一个队列。
URL示例：
http://172.16.203.197/interfaceAction/queueInterface!save.action?enterpriseId=3000008&userName=admin&password=e10adc3949ba59abbe56e057f20f883e&qno=1111&description=%E5%8C%97%E4%BA%AC&skillIds=1,2&skillLevels=4,5&id=7
说明：该示例中password = e10adc3949ba59abbe56e057f20f883e = MD5Encoder.encode('123456)，description值是经过URLEncode按“UTF-8”格式编码的，中文是“北京”。
参数说明
参数 要求 描述 备注
enterpriseId 必选 企业Id 例如:3000008
username 必选 角色名称 例如:admin
password 必选 登录密码 md5(登录密码))
id 可选 队列id 如修改时为必选
qno 必选 队列号 四位数字
description 必选 队列名 需进行UTF-8格式的URLEncode编码
queueTimeout 可选 队列超时时间 默认取值600，单位：秒，范围30-600
sayAgentno 可选 是否语音报号 boolean值，默认取值false
memberTimeout 可选 座席超时时间 默认取值25，单位：秒，范围20-60
wrapupTime 可选 整理时间 默认取值15，单位：秒，范围3-300
maxLen 可选 最大等待数 默认取值5，范围0-999
weight 可选 队列优先级 默认取值1，取值范围1-10(数值越高优先级越高)
vipSupport 可选 是否支持VIP级别 默认取值0，1表示支持,0表示不支持
strategy 可选 呼叫策略 order表示顺序,rrmemory表示轮选,fewestcalls表示平均,random表示随机,skill表示技能优先,leastrecent表示最长空闲时间.  默认取值rrmemory
musicClass 可选 等待音乐 默认公共语音库"default_moh.wav".
musicinuse 可选 等待语音播报是否开启 0表示关闭,1表示开启
announcePath 可选 等待语音播报语音文件 默认queue_thankyou(语音文件后缀名需去掉)
announceType 可选 等待语音播报类型 0表示循环间隔,1表示多于,2表示少于
announceValue 可选 等待语音播报类型的值 如 announceValue = 1.在announceType为0时表示循环间隔1秒播报,为1时表示多于1位播报,为2时表示少于1位播报.
isNext 可选 是否播报您是下一位 0表示关闭,1表示开启
skillIds 可选 所选的技能id 多个间用英文标点逗号间隔.例如:1,2,3
skillLevels 可选 所选的技能的等级(值越小技能越高) 多个间用英文标点逗号间隔.(与技能id相对应) 例如:5,1,1表示技能id为1的等级是5; 技能id为2的等级是1; 技能id为3的等级是1.
timeoutRestart 可选 呼叫超时后是否重新计时 boolean值，默认取值true
serviceLevel 可选 服务水平秒数 单位：秒，范围1-60(低于此时间内接听的认为是高水平服务)，默认取值10
retry 可选 呼叫下一个座席延期秒数 单位：秒，范围3-20 默认取值5
返回值
JSON串，格式为{result:xxx,msg:xxx}, result表示返回结果状态码， msg表示结果状态的描述。
如果成功添加队列，将返回队列id，JSON串为 {result:xxx,msg:xxx,id:xxx}。
result与msg描述定义：
result Msg
error 添加队列失败原因
success 成功添加队列
