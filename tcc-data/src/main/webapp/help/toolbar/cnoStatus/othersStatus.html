常用座席状态描述

注：以下都是在cbThisStatus(JSON)下的本座席状态

咨询

咨询成功

JSON.eventName == "consultLink"

咨询失败

JSON.name==”consultError” && JSON.eventName == “normalBusy”
注：(咨询失败以后进入正常通话状态，无具体咨询失败原因)

客户来电

咨询成功

JSON.eventName == “comeRinging” 此状态可以获取呼入客户信息

座席接听

JSON.eventName == “normalBusy” && JSON.callType== “1”

座席外呼

外呼响铃

JSON.eventName == “outRinging” (此状态包含电话详细信息)

等待客户接听

JSON.eventName == “waitLink”(此时是座席已经接听，系统开始呼叫客户，等待客户接听)

外呼客户接听

JSON.eventName == “outBusy”

挂机状态

状态为JSON.eventName == “neatenStart”

此状态包括客户呼入以及空闲时座席外呼，两者接通情况下的挂机，不论座席挂机还是客户挂机，座席的状态都是此状态。（无法区分具体挂机方）

状态为JSON.eventName == “onlineUnlink”

此状态为空闲时座席外呼，客户拒接或者客户无应答的时候，座席状态为此状态

状态为JSON.eventName == “pauseUnlink”

此状态包括座席置忙类外呼接通，座席置忙外呼客户拒接或者客户未接听。对于置忙外呼接通的情况下，无论客户或者座席哪一方挂机，座席状态都是此状态。对于座席置忙外呼客户拒接或者客户未接听，座席挂断以后，座席状态是此状态。
